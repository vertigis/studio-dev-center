VertiGIS Studio Mobile Components are styled using [XAML styles](https://docs.microsoft.com/en-us/xamarin/xamarin-forms/user-interface/styles/xaml/). However, by using VertiGIS Studio Mobile Views, you can take advantage of styled, well tested, UI components that already participate in the [application theme](configuration-theme.mdx).

## Using VertiGIS Studio Mobile Views

VertiGIS Studio Mobile Views, like the `EnhancedButton` or `EnhancedSwitch`, can ease the development of custom components with the SDK. Check out the [relevant SDK sample](https://github.com/vertigis/vertigis-mobile-samples/tree/master/Geocortex.Mobile.Samples/Geocortex.Mobile.Samples/Samples/Custom/GeocortexMobileElements) for an end to end example of their use.

### EnhancedButton

`EnhancedButton` is an extension of the [MAUI `Button`](https://learn.microsoft.com/en-us/dotnet/maui/user-interface/controls/button), with some additional properties and functionality. These include Entered and Exited events for the cursor on UWP, additional color properties, and the ability to underline the button text.

```xml
<VerticalStackLayout>
    <toolkit:EnhancedButton ImageSource="cog" ImageTintColor="MediumPurple" HandCursorOnHover="True"/>
    <toolkit:EnhancedButton Text="Hyperlink style button" BorderWidth="0" TextUnderline="True"/>
</VerticalStackLayout>
```

### EnhancedEntry

`EnhancedEntry` is an extension of [MAUI's `Entry`](https://learn.microsoft.com/en-us/dotnet/maui/user-interface/controls/entry), with some additional theming on Android.

<img src={useBaseUrl("img/mobile-enhancedEntry-android.png")} />

### EnhancedTableView

`EnhancedTableView` is an extension of the [MAUI `TableView`](https://learn.microsoft.com/en-us/dotnet/maui/user-interface/controls/tableview), with some additional color properties.

<img src={useBaseUrl("img/mobile-enhancedTableView.png")} />

### TintedImage

`TintedImage` is an extension of [MAUI's `Image`](https://learn.microsoft.com/en-us/dotnet/maui/user-interface/controls/image) which supports image tinting. Currently only supports FileImageSource sources.

```xml
<HorizontalStackLayout>
    <toolkit:TintedImage TintColor="DarkGoldenrod" Source="{Binding CogIcon, Converter={StaticResource IconConverter}, ConverterParameter='cog'}"/>
    <toolkit:TintedImage TintColor="MediumPurple" Source="{Binding CogIcon, Converter={StaticResource IconConverter}, ConverterParameter='cog'}"/>
</HorizontalStackLayout>
```

<img src={useBaseUrl("img/mobile-tintedImage.png")} />

## Using Theme Colors in Custom Components

If you need to use plain Views when building your component, the various [theming colors](configuration-theme.mdx#theme-color-reference) can be used in XAML styling.

```xml
<Style x:Key="InnerDetailsFrameStyle" TargetType="Frame">
  ...
  <Setter Property="BackgroundColor" Value="{DynamicResource SecondaryBackgroundColor}"></Setter>
  ...
</Style>
```

## Using Built-in Icons

The following example demonstrates a component that uses VertiGIS Studio Mobile UI components and an icon from the VertiGIS Studio Icons library.

<Tabs
  defaultValue="component"
  values={[
    { label: 'Component', value: 'component', },
    { label: 'UI', value: 'ui', },
  ]
}>
<TabItem value="component">

```cs
class CustomComponent : ComponentBase
{
    public CustomComponent()
    {
    }

    protected override VisualElement Create(XNode node)
    {
        return new EnhancedButton()
        {
            Text = "Pause Icon",
            // highlight-next-line
            ImageSource = "playback_pause",
            Margin = new Thickness(2)
        };
    }
}
```
