VertiGIS Studio Mobile was designed to be fully internationalizable on all levels of the product, from app config to custom code. This internationalization is handled by .NET MAUI.

## Specifying a Default Language

Microsoft describes how to [specify a default language for your app](https://learn.microsoft.com/en-us/dotnet/maui/fundamentals/localization?view=net-maui-9.0#specify-the-apps-neutral-language).

## Adding New Cultures for Existing Language Strings

The [Quickstart](https://github.com/vertigis/vertigis-mobile-quickstart) project comes with multiple `AppResources` files in the [root of the platform agnostic project](https://github.com/vertigis/vertigis-mobile-quickstart/tree/master/App1/App1) to provide an example of adding a culture to your application. You can also check out the [MAUI Documentation on Localization](https://learn.microsoft.com/en-us/dotnet/maui/fundamentals/localization).

## Relevant SDK Sample

Check out the relevant VertiGIS Studio Mobile SDK Sample:
